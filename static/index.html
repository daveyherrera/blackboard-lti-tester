Setup Guide
Five steps to your first LTI 1.3 launch
1
Install & Start the Local Server
›
  1. Clone or download this repo to your Mac.
  2. Open Terminal and cd into the project folder.
  3. Run the start script:
./start.sh

This will:

  • Create a Python virtual environment and install dependencies
  • Start the FastAPI server on port 8080
  • Start an ngrok HTTPS tunnel and print your public URLs
  • Open this dashboard in your browser

Requires python3 and ngrok. Install ngrok with:
brew install ngrok/ngrok/ngrok then ngrok config add-authtoken <TOKEN>

2
Register Your Application in the BB Developer Portal
›
  1. Go to developer.blackboard.com and sign in.
  2. Navigate to My Apps → Register.
  3. Fill in application details, then under LTI 1.3 settings enter:
  • Login Initiation URL: your ngrok URL + /oidc-login
  • Tool Redirect URL(s): your ngrok URL + /redirect
  • Tool JWKS URL: your ngrok URL + /jwks

After saving, copy the Application ID (this is your Client ID) and the OIDC Auth URL and BB JWKS URL shown on the app page.

3
Configure This Tool
›
  1. Click Settings in the left sidebar.
  2. Enter the Client ID from the Developer Portal.
  3. Confirm the OIDC Auth URL and BB JWKS URL match your portal values.
  4. Click Save Settings.

The Dashboard should now show Config as Complete.

4
Register the LTI Tool in Blackboard Learn
›
  1. Go to Administrator Panel → Integrations → LTI Tool Providers.
  2. Click Register LTI 1.3/Advantage Tool.
  3. Enter your Client ID (Application ID from the Developer Portal) and click Submit.
  4. Learn fetches and auto-populates the tool's settings from the Developer Portal — they display as read-only. Review and click Submit.
  5. Set Approval Status to Approved.
  6. Note the Deployment ID shown on the tool's detail page — enter it in this tool's Settings page.

You can also configure Institution Policies on this page to control what user data Learn sends in the launch (name, email, role) and whether the tool can access the Grade and Membership services.

5
Create a Placement
›

Placements control where and how the tool appears in Blackboard Learn. You need at least one to launch. Create one per use case you want to test.

  1. From the tool's detail page click Manage Placements → Create Placement.
  2. Fill in the fields:
Field Value
LabelName users see in courses (e.g. BB LTI Tester)
HandleUnique identifier — alphanumeric, underscores, dashes only (e.g. bb-lti-tester)
Type Course tool — regular LTI launch, appears under course tools
Deep Linking content tool — instructor picks content to embed; triggers a LtiDeepLinkingRequest launch
Target Link URIhttps://daveyherrera.github.io/blackboard-lti-tester/
Allow Student AccessEnable to test Learner role launches

Tip: Create one Course tool placement and one Deep Linking content tool placement so you can test both message types from the same registered application.

6
Launch the Tool from a Course
›
  1. Enter a course in Blackboard Learn as an Instructor or Student.
  2. Add the tool to a content area: Build Content → [your placement name].
  3. Click the tool link — Learn initiates the OIDC login flow.
  4. Watch the Launches page here — the JWT appears within seconds.
  5. Click any launch to inspect the full payload: user info, roles, context, services, and Blackboard extensions.
  6. To submit grades, fetch the roster, or respond to Deep Linking requests, use the LTI Lab — the interface that loads inside the Blackboard iframe when you launch the tool.

Troubleshooting: If the launch doesn't appear, check that ./start.sh is still running and the URLs in the Developer Portal match your current ngrok hostname. ngrok URLs change on every restart — update them each time.
