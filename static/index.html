Setup Guide
Get LTI launches flowing in 5 steps
1
Install & Start
▶

Clone the repo and run the start script:

git clone <your-repo-url> bb-lti-tester
cd bb-lti-tester
chmod +x start.sh stop.sh
./start.sh

This will:

  • Create a Python virtual environment
  • Install all dependencies
  • Generate RSA keys automatically
  • Start the FastAPI server on port 8080
  • Start ngrok tunnel (if installed)
  • Open your browser to the dashboard
2
Register in Blackboard Developer Portal
▶

Go to developer.blackboard.com → My Apps → Create App

Fill in these fields:

  • Application Name: BB LTI Tester (or any name)
  • OIDC Login Initiation URL: —
  • OIDC Redirect URL(s): —
  • JWKS URL: —

After creating, you'll receive a Client ID. Copy it — you'll need it in Step 3.

3
Configure This Tool
▶

Go to and enter:

  • Client ID — from Developer Portal
  • OIDC Auth URL — https://developer.blackboard.com/api/v1/gateway/oidcauth
  • Blackboard JWKS URL — https://developer.blackboard.com/api/v1/management/applications/keys

Click Save. The Config status badge on the dashboard will turn green.

4
Register in Blackboard Learn
▶

In your Blackboard Learn instance:

  • Go to System Admin → LTI Tool Providers
  • Click Register LTI 1.3 Tool
  • Enter your Client ID from the Developer Portal
  • Approve the tool — you'll get a Deployment ID
  • Copy the Deployment ID and optionally save it in Settings
5
Place & Launch the Tool
▶

In a Blackboard course:

  • Edit a content area → Build Content → LTI Tool
  • Select your registered tool
  • Click Submit
  • Navigate to the content item and click the tool link
  • Blackboard will call your /oidc-login endpoint
  • You'll be redirected back and the launch will appear in the page
